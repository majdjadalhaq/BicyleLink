
Title: Core Identity Features & Security Updates

Description:
Hi team,

I finished the implementation for the core identity infrastructure. Here is what I changed in this PR:

Backend Changes:
- I updated the User model: added `isVerified` (default false) and `agreedToTerms` (must be true).
- If a user is not verified, they cannot login (returns 403 Forbidden).
- Passwords are now hashed securely using bcrypt.
- I cleaned up some code in standardizing ports and added CORS settings.

Frontend Changes:
- I updated the Signup page.
- Added a required checkbox for Terms of Service.
- Fixed the data payload sent to the backend (removed confirmPassword, added name).

Testing:
- Use Cypress to test the signup flow and login guard.
- Updated unit tests to work with the changes.

Note: I included a folder called `pr2_backup` which has my work for the next feature (email verification), so ignore that for now.

Let me know if you have any feedback!
