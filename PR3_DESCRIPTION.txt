
Title: Verification Enhancements (Resend Code & Security)

Description:
Hi team,

I implemented the "Resend Verification Code" feature with production-grade security controls.

What's New:
- **Resend Button**: Users can request a new code if they didn't receive the first one.
- **Countdown Timer**: 60-second cooldown on the frontend (and enforced on backend).
- **Rate Limiting**: Strictly limited to 3 resends per hour to prevent spam.
- **Security**:
  - `verificationCodeLastSentAt`, `resendCount`, `windowStart` added to User schema.
  - Anti-enumeration responses (always returns success message even if user not found).
  - Code invalidation on new request.

Backend:
- `POST /users/resend-code` endpoint.
- Unit/Integration tests added (`test-resend.js`) covering edge cases (cooldown, rate limit cap, window reset).

Frontend:
- Updated `VerifyCode` page with timer logic and error handling (429 Too Many Requests).

Testing:
- Verified backend logic using `test-resend.js` (All scenarios passed).
- Manual verification on frontend pending (requires Mailtrap check).

This PR builds on PR #2.
